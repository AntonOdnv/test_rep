### Подключаемся к Git
### Генерируем ключ для работы по SSH (вместо <имя пользователя> укажите свое имя/почту)
> cd ~/.ssh
> ssh-keygen -t rsa -b 2048 -C "<имя пользователя>" 
### В диалоговом окне введите название файла ключа. 
### passphrase можно оставить пустым, чтобы потом не пришлось вводить пароль каждый раз

id_rsa - это секретный ключ, его никогда не передаем никому! 
Если указали другое имя - вместо id_rsa будет имя вашего ключа. Далее в примерах id_rsa - имя ключа (заменить на свое, если меняли).
id_rsa.pub - это открытый (публичный) ключ.
Публичным ключом можно смело делиться и передавать в открытом виде в другие сервисы и инструменты.

### Копируем открытый ключ  из терминала, вставляем его в https://github.com/settings/ssh/

### Чтобы настроить подключение из-под чистого Linux к Git, 
### нужно сначала запустить ssh_agent:
> eval "$(ssh-agent -s)"
### Затем добавляем ключ в агент (если меняли имя, вместо id_rsa укажите имя вашего ключа):
> ssh-add ~/.ssh/id_rsa

### Проверяем подключение к гиту командой:
> ssh -T git@github.com

Если увидели фразу вроде "Hi <ваше_имя>! You've successfully authenticated... " - значит все хорошо.
Если нет - открываем любой чат Gen AI, копируем ошибку, ищем решение сами или задаем вопросы.

### Идем в папку, где будем работать с проектом. Например, в корень командой > cd ../ 
### Клонируем туда наш репозиторий
> git clone <your_ssh_link>

### Инициализируем и настраиваем git
git init
git config --global user.email "you@example.com"
git config --global user.name "Your Name"

### Теперь Git должен без проблем работать из консоли
